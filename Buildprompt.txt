# Build a web application called **KUER Studio**

KUER Studio is an AI-powered branded QR code design platform that allows users to enter a URL and generate visually stunning, fully scannable QR codes in a wide variety of brand styles.

The goal is not simply to generate a QR code, but to create a branded marketing asset suitable for websites, social media, print, packaging, events, podcasts, AR experiences, videos, and campaigns.

The application should be built mobile-first with a premium AI-design-studio aesthetic. Think Apple-inspired clarity, dark premium panels, subtle glassmorphism, soft shadows, excellent spacing, fluid micro-interactions, and professional export workflows.

# Critical Technical Requirement

The QR code must always remain functional.

The platform must never generate beautiful artwork without validating scanability.

Every generated variant must pass through a validation pipeline.

## Validation Pipeline

* Render QR Variant
* Decode using at least one QR decoder
* Verify decoded URL exactly matches original URL
* Measure contrast score
* Verify quiet zone
* Verify finder patterns
* Verify minimum resolution
* Verify export suitability for print or digital
* Generate scanability score (0–100)

---

# Validation Statuses

* Validated
* Needs Contrast Improvement
* Quiet Zone Too Small
* Finder Pattern Damaged
* Low Scan Confidence
* Not Scannable

---

# Auto Repair Suggestions

If validation fails, automatically provide:

* Increase Contrast
* Restore Finder Patterns
* Add Quiet Zone
* Simplify Artwork
* Reduce AI Overlay
* Generate Safer Variant

# QR Protection Mask System

The following areas must always be protected:

* Top-left finder pattern
* Top-right finder pattern
* Bottom-left finder pattern
* Alignment patterns
* Timing patterns
* Quiet zone
* Critical data modules depending on QR density

AI should decorate around and between QR modules but must not destroy scan-critical structures.

Maintain sufficient module contrast at all times.

# Validator Interface

```ts
validateQrImage(imageBuffer, expectedUrl)
```

Returns:

```ts
{
  isScanable: boolean,
  decodedUrl: string | null,
  urlMatches: boolean,
  score: number,
  checks: {
    contrast: "pass" | "warning" | "fail",
    quietZone: "pass" | "warning" | "fail",
    finderPatterns: "pass" | "warning" | "fail",
    resolution: "pass" | "warning" | "fail"
  },
  recommendations: string[]
}
```

Do not display false validation claims.

Use:

* "Demo Validation"
* "Validation Simulation"

until a real decoder is active.

Once a real decoder is active:

* Display "Validated"
